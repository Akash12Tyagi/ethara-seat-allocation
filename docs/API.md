# API Reference

Full interactive docs (request/response schemas, try-it-out) are auto-generated by FastAPI at **`/docs`** (Swagger UI) and **`/redoc`** once the backend is running. This file is a quick reference; treat `/docs` as authoritative.

Base URL (local): `http://localhost:8000`

## Employees

| Method | Path | Description |
|---|---|---|
| POST | `/employees` | Create employee. Body: `{name, email, department, role, joining_date, project_id?, employee_code?}`. 409 on duplicate email. |
| GET | `/employees` | Paginated list. Query: `search, project_id, department, status, seat_status, page, page_size` |
| GET | `/employees/pending-allocation` | Active employees with no active seat |
| GET | `/employees/{id}` | Single employee, includes seat + project if allocated |
| PUT | `/employees/{id}` | Partial update |
| DELETE | `/employees/{id}` | Soft-deactivate (sets status=inactive, releases any active seat) |
| POST | `/employees/csv-upload` | Bulk create from CSV (`name,email,department,role,joining_date,project_name`) |

## Projects

| Method | Path | Description |
|---|---|---|
| POST | `/projects` | Create. Body: `{name, description?, manager_name?, status?}`. 409 on duplicate name. |
| GET | `/projects` | List all, with `employee_count` |
| GET | `/projects/{id}/employees` | Employees on this project |

## Seats

| Method | Path | Description |
|---|---|---|
| POST | `/seats` | Create. Body: `{floor, zone, bay, seat_number, status?}`. 409 on duplicate floor+zone+seat_number. |
| GET | `/seats` | Paginated list. Query: `floor, zone, status, project_id, page, page_size` |
| GET | `/seats/available` | Available seats, optionally filtered by `floor`/`zone` |
| GET | `/seats/{id}` | Single seat |
| POST | `/seats/allocate` | `{employee_id, seat_id?, preferred_floor?, preferred_zone?}`. Omit `seat_id` for auto-select. 409 if employee already has a seat, seat unavailable, or none available anywhere. |
| POST | `/seats/release` | `{employee_id}` or `{seat_id}`. 404 if no active allocation. |

## Dashboard

| Method | Path | Description |
|---|---|---|
| GET | `/dashboard/summary` | Totals: employees, seats, occupied/available/reserved/maintenance, pending allocation |
| GET | `/dashboard/project-utilization` | Per-project employee count, allocated seats, utilization % |
| GET | `/dashboard/floor-utilization` | Per-floor seat counts by status, occupancy % |
| GET | `/dashboard/recent-allocations` | Last N allocate/release events |

## AI Assistant

| Method | Path | Description |
|---|---|---|
| POST | `/ai/query` | `{query, employee_email?}` → `{answer, intent, data}`. See README "AI Assistant" section for supported query types. |

Example:
```bash
curl -X POST http://localhost:8000/ai/query \
  -H "Content-Type: application/json" \
  -d '{"query": "Where is employee Amit seated?"}'
```
```json
{
  "answer": "Amit Sharma is seated on Floor 2, Zone B, Bay 4, Seat B4-23. Assigned to Project Talos.",
  "intent": "employee_seat_lookup",
  "data": {"employee_id": 1}
}
```

## Auth

No hard auth in this build — see `app/deps.py` for the lightweight `X-Role: admin|hr|employee` header used to distinguish HR/Admin views from employee self-service in the frontend. This is not a security boundary; see REQUIREMENTS.md "Known Gaps" for the reasoning.
